# 프로젝트 진행 상황 및 로드맵 (BHM Overtime)

## ✅ 최근 완료된 작업 (Phase 1: 핵심 기능 오류 수정 및 안정화)

1. **공휴일 API 비동기 처리 개선 (app.js)**
   - 문제: 공휴일 외부 API(`apis.data.go.kr`) 응답이 네트워크 문제로 지연되거나 실패할 경우, 캘린더 전체 렌더링이 중단되고 화면이 백지로 표시되는 심각한 결함이 있었습니다.
   - 해결: `refreshOtCalendar`와 `refreshLvCalendar` 함수를 **이중 렌더링(Double Rendering) 아키텍처**로 재설계했습니다. 이제는 로컬 캐시 데이터를 이용해 달력 UI 프레임을 1차로 즉시 그리고, 백그라운드에서 공휴일 데이터를 가져온 뒤 2차로 마커만 깜빡임 없이 업데이트합니다.

2. **초기 렌더링 충돌 (빈 화면 버그) 수정 (app.js)**
   - 문제: 페이지 최초 로드 시 DOM 요소가 누락되어 캘린더 렌더링 함수(`initOvertimeTab()`) 자체가 호출되지 않는 버그가 있었습니다. 원인은 사용자가 화면 단순화를 위해 `index.html`에서 불필요한 UI(`wJobType`, `psPayTable` 등)를 제거했음에도 불구하고, 초기 로딩(`DOMContentLoaded`)을 담당하는 JS 이벤트 리스너가 이를 강제로 참조하려다 발생한 `TypeError`였습니다.
   - 해결: `app.js`의 `updateGrades` 함수 등에 방어 로직(Null Check)을 도입하여, HTML 요소가 존재하지 않을 때 스크립트가 강제 종료되지 않고 안전하게 초기화를 마칠 수 있도록 처리했습니다.

3. **버그 픽스 사항 Git 저장소 반영**
   - 앞선 모든 코드 변경사항들을 종합하여 `Fix: calendar initial load crash and API delay rendering bugs` 커밋 메세지와 함께 Git에 병합 완료했습니다.

## 🔜 앞으로 진행할 작업 (Phase 2 & 3: UX/UI 강화 및 백엔드 확장)

사용자의 목표에 맞춰 시간외·온콜 기능을 더욱 직관적으로 만들고 제품의 스케일을 확장하는 단계입니다.

### 🎯 1단계: 시간외·온콜 UX/UI 개선 (직관성 강화)
- **시간 직접 입력 방식 정착:** 이전에 존재했던 단축 버튼들을 완전히 제거하고, 직관적인 시작/종료 시간 직접 입력 폼의 동선(Flow)을 최적화합니다.
- **시각적 계층 구조(Visual Hierarchy) 정리:** 수당 금액 등 중요 정보가 가장 먼저 눈에 띄도록 대시보드 레이아웃과 폰트 크기, 대비를 개선합니다.
- **불필요한 인지 부하 감소:** 입력 패널과 달력 간의 상호작용 피드백(선택, 저장, 삭제 등)을 더욱 명확히 표시합니다.

### 📦 2단계: Supabase 데이터베이스 연동
현재 LocalStorage에 의존하던 데이터 관리 방식을 클라우드 백엔드로 마이그레이션합니다.
- **Supabase 초기 설정:** 프로젝트 생성, API Key 발급, 클라이언트 연동 환경을 설정합니다.
- **사용자 인증 (Auth):** 사용자별 데이터 분리 및 보안을 위한 로그인/회원가입 절차(또는 세션 관리)를 구현합니다.
- **데이터 스키마 설계 및 동기화:** `PROFILE`(개인정보), `OVERTIME`(시간외 기록), `LEAVE`(휴가 기록) 데이터를 위한 테이블을 생성하고, 프론트엔드 API 호출을 통해 기존 로직과 매끄럽게 연동합니다.

### 🛡️ 3단계: Impeccable 플러그인을 통한 품질 보증 및 마감
사전에 연동된 AI 에이전트 전용 `Impeccable` 플러그인 워크플로우를 가동하여 제품 퀄리티를 전문가 수준으로 끌어올립니다.
- **`/teach-impeccable`:** 프로젝트의 디자인 철학과 사용자 컨텍스트를 문서화하여 AI 개발의 나침반으로 삼습니다.
- **품질 진단 (`/audit`, `/critique`):** 작성된 프론트엔드의 웹 접근성(A11y), 성능 모틀넥, 반응형 이슈 및 UX 사용성 결함을 꼼꼼히 진단받고 개선안을 즉시 반영합니다.
- **스타일 & 상호작용 고도화 (`/normalize`, `/polish`, `/delight`):** 여백, 컬러 시스템, 정렬 등 디자인 시스템을 일관되게 적용하고, 부드러운 마이크로 애니메이션이나 빈 화면(Empty State) 디자인을 추가하여 사용자 만족도와 탄력성(Resilience)을 극대화합니다.
