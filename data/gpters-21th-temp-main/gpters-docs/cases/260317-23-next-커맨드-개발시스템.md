## 소개

개발하던 갤럭시 링 연동 앱의 코드베이스의 구현 상태와 보안 상태를 점검하고, 클로드 코드를 실행한 후 /next 라는 슬래시 커맨드만 입력하면 자동으로 다음에 구현할 것을 구현하는 개발 시스템을 컨텍스트 엔지니어링으로 구현했습니다. 바이브코딩하면서 할 일은 보고서를 읽는 것과 실제로 실행해서 테스트하는 것 뿐으로 만들고 싶었습니다.

## 진행 방법

먼저 Agent Teams 이용하여 개발하다 중단된 앱의 개발 상태와 보안 상태를 점검해서 문서로 남겼습니다.

![\"\"](https://tribe-s3-production.imgix.net/GnwD69ISgHzNxEgdpsMHA?auto=compress,format)

그. 다음에 클로드 코드에게 컨텍스트 엔지니어링을 적용해서 Skill, Hook, Agent, CLAUDE.md, rules를 설계해달라고 했습니다.

이 프로젝트에 특화된 시스템이며, 앞서 작성한 두 문서를 참고했습니다.

![\"\"](https://tribe-s3-production.imgix.net/Qzi5TV0f8PUOizcJIGb65?auto=compress,format)

그 다음에, 앱의 개발 상태와 이 개발 시스템 설계를 참고해서 전체 시스템 완성까지의 to-do-list를 만들었습니다.

to-do-list 한 항목의 크기는 프롬프트 한 번에 끝낼 수 있을 정도로 작은 걸로 헀습니다.

98개 항목이 나오네요. 요약입니다.

![\"\"](https://tribe-s3-production.imgix.net/S6787xjBDLTAoZIaefNrW?auto=compress,format)

또, 이 작업 목록을 토대로 CLAUDE.md 파일 설계를 좀 수정하고

progressive disclosure를 적용해서 항상 필요하지 않은 내용은 필요 시마다 참조하도록 했습니다.

![\"한국판](https://tribe-s3-production.imgix.net/pYq8D7WqXBtpcIqlr5oyV?auto=compress,format)

그리고 이제 설계대로 Skill, Hook, Agent, CLAUDE.md를 만들었습니다.

이 설계는 실행 가능한 수준으로 가이드 문서를 만들었기 때문에

가이드 문서에 기술된 순서대로 실행해달라고 했어요.

그렇지만 결과가 쏙 마음에 들지는 않아서(내용이 좀 부실했습니다.)

스킬, 훅, 에이전트는 따로 보강했습니다.

그 다음에, 하고자 했던 custom slash command를 만들기 위한 프롬프트를 썼습니다.

```
/Users/rose/Desktop/flutter_project/galaxy_ring_project/galaxy-ring-app-flutter/docs/implement \n  ation_todo.md                                                                                  \n  ---                                                                                            \n  여기의 작업을 하나씩 가져와서 설계 -> 구현 -> 테스트  -> 보안 점검 -> git push를 하도록 하는   \n  custom slash command A를 하나 만들어 줘. 만약 API Key를 발급받는 것처럼 내가 해결해야 할 일이  \n  있으면 해야 하는 일을 step-by-step 으로 정리해서 @docs/user-guide에 문서로 만들도록 해.   \n  그리고 custom  slash command A가 완료되면\n  자세한 보고서를 @docs/step-report에 꼭 남기도록 해.     \n  그리고 전체 작업을 보면서 custom slash command A를 하나씩 실행하고, 현재 어디까지 실행했는지   \n  파악하는 custom slash command B도 만들어 줘. 나는 custom slash command B만 실행하면, 모든      \n  시스템 구현이 실행할 떄마다 이어서 진행되도록 해 줘. 내 설계는 참고로 하고, 더 좋은 아이디어가 \n   있으면 그걸로 진행해 줘. \n   Task management  system 써서 체계적으로 진행하고 필요하면 Agent teams 사용해.
```

이렇게 /step과 /next라는 커스텀 슬래시 커맨드를 만들었습니다.

![\"\"](https://tribe-s3-production.imgix.net/kGQ73KWYRgH9M4PzrMaux?auto=compress,format)

저는 이렇게 클로드 코드를 켜고 항상 /next 만 입력하면 됩니다.

그리고 보고서를 확인하고, 기능 점검을 위해 실행해보구요.

![\"\"](https://tribe-s3-production.imgix.net/TmSeU2xwzGVApmuODR34l?auto=compress,format)

/next 입력해서 기능 하나 구현했습니다. github commit 까지 잘 되네요.

## 결과와 배운 점

컨텍스트 엔지니어링으로 얼마나 개발 시스템을 편하게 할 수 있는지 알게되었습니다.

이제, Openclaw로 Claude 사용량 체크를 해가면서 자동으로 /next 실행하고,

제가 확인해야 할 사항이 생기면 리포트를 생성하도록 했으니

리포트가 생성되면 메신저 알람을 주고,

매일의 작업을 보고서로 만들어 메신저로 보내주는 시스템을 구축하고 싶습니다.

제가 할 일은 이제 메신저 보고, 매일 한 번 앱 실행해보는 게 되겠네요.

문제가 생기면 문제 해결하구요.

이 시스템으로 얼마나 가능할 지 기대가 됩니다.
