## 📝 한줄 요약

Airtable에 쌓이는 AX 게시글을 매주 금요일 Slack에 자동 정리해서 보내주는 봇을, Claude Code로 바이브코딩하여 만들었습니다.

## 🎯 이런 분들께 도움돼요

- Airtable에 데이터가 쌓이는데, Slack이나 다른 채널에 자동으로 보내고 싶은 분
- 사내 커뮤니티 게시글을 팀원들에게 정기적으로 공유하고 싶은 분
- AI 코딩 도구로 간단한 자동화 봇을 만들어보고 싶은 분

## 😫 문제 상황 (Before)

커뮤니티에 AX 게시글이 꾸준히 올라오고 있었는데, 팀원들이 이를 보는 데에 어려움이 있었습니다.

게시글이 올라와도 각자 바쁘다 보니 확인을 안 하고, 좋은 글이 묻히는 일이 반복됐어요. \\"이번 주에 이런 글 올라왔어요\\"라고 누군가 수동으로 정리해서 공유하면 좋겠다는 생각은 있었지만, 그걸 매주 챙기는 건 또 다른 일이었습니다.

결국 주기적으로 모아서 알려주는 자동화가 필요하다고 판단했고, Claude Code로 직접 만들어보기로 했습니다.

## 🛠️ 사용한 도구

- **AI 도구**: Claude Code
- **데이터**: Airtable (게시글이 쌓이는 테이블)
- **알림**: Slack Incoming Webhook
- **배포/스케줄링**: Vercel Cron Job (매주 금요일 09:30 자동 실행)

## 🔧 작업 과정

### 1\. \\"봇 만들어줘\\" 한 문장으로 시작

처음에는 구체적인 기술 스펙 없이, 하고 싶은 걸 그대로 말했습니다.

> Airtable에서 주간 신규 AX 게시글을 수집하여 매주 금요일 09:30 KST에 Slack #00-ai-workflow 채널에 자동 포스팅하는 봇을 만들어줘

이 한 문장으로 Claude Code가 만들어준 것들:

- Airtable에서 데이터를 가져오는 연동 로직
- 금요일~금요일 기준으로 \\"이번 주\\" 범위를 계산하는 날짜 로직
- 게시글 요약본에서 핵심 내용을 자동으로 뽑아내는 파서
- Slack에 보기 좋게 정리해서 보내주는 포맷터
- 매주 금요일 09:30에 자동 실행되는 스케줄링 설정

Airtable에서 데이터를 가져오고, 정리해서, Slack에 보내는 전체 파이프라인이 한 번에 나왔습니다. 파일 6~7개가 한꺼번에 생성됐는데, 각각의 역할이 깔끔하게 분리되어 있었어요.

추가적으로 Slack bot 셋팅, Vercel Cron Job 셋팅은 대부분 클로드코드가 시키는 대로 했더니 잘 진행되었지만, 주의할 점은 봇 웹훅 생성 전에 Display Name(Bot Name)을 설정해두어야 웹훅 생성이 가능하다는 점입니다.

![\"앱](https://tribe-s3-production.imgix.net/iXVOY625rWLaXCvGj5mIA?auto=compress,format)

### 2\. \\"이거 AX 게시글만 나와야 하는데?\\" — 문제를 말하면 바로 해결

첫 번째 결과를 돌려보니, AX 게시글 외에 다른 게시글도 섞여서 나왔습니다. Airtable 테이블에 여러 종류의 글이 같이 있었거든요.

> 태그='지피터스는사내AX중' 필터 추가하고, 닉네임/요약본 포맷도 개선해

이렇게 말하니까 Airtable 쿼리에 태그 필터를 추가하고, 닉네임 표시 방식과 요약본 포맷까지 한 번에 개선해줬습니다.

### 3\. \\"닉네임이 이상하게 나와요\\" — 현상만 말하면 AI가 원인까지 찾아줌

포맷을 개선한 후에도 닉네임이 깨져서 나오는 문제가 있었습니다. 배열 형태(`[\"홍길동\"]`)로 표시되는 거예요.

원인이 뭔지 몰라서 그냥 \\"닉네임이 배열로 나온다\\"고 말했는데, Claude Code가 알아서 원인을 파악했습니다. Airtable에서 닉네임이 linked record(연결된 레코드)로 되어 있어서 배열로 넘어오는 거였고, 첫 번째 값만 꺼내도록 처리해줬어요.

이때 \\"디버깅도 AI가 해주는구나\\" 싶었습니다. 코드를 모르더라도 **현상을 정확하게 설명하면** AI가 원인 파악부터 수정까지 해주더라고요.

### 4\. 마무리 — 번호 붙이고 알림 켜기

기본 기능이 완성된 후, 사용성을 높이기 위해 마지막 요청을 했습니다.

> 게시글에 번호를 붙이고, @channel 멘션으로 알림이 가게 해줘

이렇게 해서 게시글마다 순번이 붙고, 메시지가 올라올 때 채널 전체에 알림이 가도록 완성했습니다.

## 💬 이 과정에서 배운 AI 활용 팁

### 효과적이었던 것

1. **결과가 이상하면 현상을 구체적으로 말하기** — \\"닉네임이 배열로 나온다\\"처럼 눈에 보이는 현상을 정확히 알려주면, AI가 원인 파악부터 수정까지 한 번에 해줍니다. 코드를 몰라도 됩니다.
2. **처음 요청할 때 전체 그림 알려주기** — \\"Airtable에서 가져와서 Slack에 보내줘\\"처럼 입력-출력을 한 문장으로 설명하면, 중간 과정을 AI가 알아서 설계합니다.

### 이렇게 하면 안 돼요

1. **\\"안 돼요\\"만 말하기** — \\"안 돼요\\" 대신 \\"닉네임이 배열로 나와요\\", \\"AX 외 글도 섞여요\\"처럼 뭐가 어떻게 안 되는지 말해야 합니다.
2. **한 번에 너무 많이 바꾸기** — 기본 기능 먼저 확인하고, 필터링 → 포맷 개선 → 번호/멘션 순으로 단계별로 요청하는 게 결과가 좋았습니다.

## 🌍 다른 업무에 적용한다면?

- **사내 공지 자동화**: 특정 채널의 공지사항을 다른 채널에 요약해서 전달
- **주간 리포트 자동화**: Airtable/Notion에 쌓이는 업무 데이터를 매주 정리해서 Slack에 공유
- **고객 리뷰 모니터링**: 새로 들어온 고객 피드백을 매일 팀 채널에 요약 전달

어디에 데이터가 쌓이고, 어디로 알림을 보내고 싶은지만 정하면 같은 구조로 만들 수 있습니다.

## 🚀 앞으로의 계획

사용하면서 개선점을 찾아갈 예정입니다. 실제로 팀원들이 어떤 게시글에 반응하는지, 어떤 포맷이 더 읽기 편한지 지켜보면서 조금씩 다듬어가려 합니다.
