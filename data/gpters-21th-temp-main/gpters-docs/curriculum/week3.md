# Week 3: 스킬 만들기 + 고급 활용 ("나만의 스킬 만들기")

## 목표

- 반복 작업을 스킬(SKILL.md)로 만드는 방법 배우기
- 프로젝트에서 반복되는 패턴을 찾아 자동화하기
- 다른 사람의 스킬을 참고해서 내 것으로 응용하기

## 모임 아젠다 (1시간)

1. 지난 주 과제 공유 — 첫 구현 결과 데모 (15분)
2. 스킬이란? 라이브 데모 — 기존 스킬 구조 살펴보기 + 새 스킬 만들기 (20분)
3. 실습: 각자 프로젝트에서 반복 패턴 찾고 스킬 초안 만들기 (15분)
4. 스킬 아이디어 공유 + 피드백 (10분)

## 사용 스킬

- /skill-creator — 스킬 SKILL.md를 자동으로 생성해주는 도구
- /gpters-thinking-partner — "어떤 걸 스킬로 만들면 좋을까?" 함께 탐색
- /gpters-skill-updater — 만든 스킬을 개선하고 다듬기

## 과제

1. `/skill-creator`로 나만의 스킬 1개 만들기 (SKILL.md 생성)
2. 만든 스킬을 실제로 3번 이상 사용해보고 개선점 찾기
3. `/gpters-writer`로 "나만의 스킬 만들기" 사례글 작성 → GPTers 게시

## 팁

- 스킬은 "매번 같은 말을 반복하게 되는 것"에서 시작하세요.
- 예시: 매번 "이 코드 리뷰해줘, 포맷은 이렇게..." → `/code-review` 스킬로 만들기
- 처음에는 간단하게 시작하고, 쓰면서 조금씩 다듬는 게 핵심입니다.
