# GPTers Writer

이번 주 Claude Code 작업을 GPTers 사례글로 변환합니다.
3턴 대화 → 결과 미리보기 → HITL 확인 → 문서화.

## Quick Start

```text
/gpters-writer              # 기본 실행
```

## Workflow

### Phase 0: 세션 자동 스캔 (선행)

`~/.claude/projects/` 에서 현재 프로젝트 세션을 자동 스캔.
- 세션 있음 → 요약을 Turn 1 질문에 힌트로 반영: "Claude Code 세션을 확인해보니 {작업 요약}을 하신 것 같아요!"
- 세션 없음 → 힌트 없이 Turn 1 진행

### Turn 1: 뭘 했나요?

AskUserQuestion:
- "이번 주에 Claude Code로 어떤 작업을 했나요?"
  - 새로운 기능/스킬을 만들었어요
  - 기존 코드를 수정/개선했어요
  - 문서나 콘텐츠를 만들었어요
  - 기타 (직접 설명)

### Turn 2: 전에는 어땠나요?

AskUserQuestion:
- "이 작업을 Claude Code 없이 했다면 어땠을까요?"
  - 훨씬 오래 걸렸을 거예요 (시간 단축)
  - 아예 못 했을 거예요 (불가능 → 가능)
  - 품질이 많이 떨어졌을 거예요 (품질 향상)
  - 큰 차이 없었을 수도 (솔직한 평가)

### Turn 3: 하이라이트는?

AskUserQuestion:
- "가장 인상적이었던 순간은?"
  - 프롬프트 한 줄로 뚝딱 해결된 순간
  - 시행착오 끝에 성공한 순간
  - 예상 못한 결과가 나온 순간 (좋든 나쁘든)
  - 특별한 건 없었지만 전체적으로 편했음

## Result: 사례글 미리보기 (채팅 출력)

3턴 답변 + 세션 스캔(가능 시) 종합하여 채팅에 미리보기:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📝 사례글 미리보기
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

제목 후보:
  1. "{문제} → {해결} 이야기"
  2. "{도구/방법}으로 {결과} 달성"
  3. "{감정/상황}, AI한테 맡겨보니"

─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─

## 배경
{Turn 2 기반: 이전 상황과 불편함}

## 과정

- 사용한 기능: {감지된 기능}
- 프롬프트 예시: {세션에서 추출 또는 추정}

## 결과
{Turn 2 기반: before/after}

## 배운 점
{Turn 3 기반: 인사이트}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 세션 보강

Phase 0에서 스캔한 세션이 있으면 DEVLOG 요약을 미리보기에 반영 (수정 파일, 턴 수, 작업 내용).
없으면 Turn 1-3 답변만으로 작성.

## HITL: 확인 + 문서화

AskUserQuestion:
- "이 사례글로 어떻게 할까요?"
  - 이대로 저장 → 문서화 진행
  - 제목 바꿀래요 → 제목 선택 후 저장
  - 내용 수정할 부분이 있어요 → 피드백 반영 후 다시 미리보기
  - 처음부터 다시 할래요 → Turn 1로

### 문서화

`my/cases/YYMMDD-{slug}.md` 에 저장.

### Slug 생성 규칙

파일명: `YYMMDD-{slug}.md`
- YYMMDD: 스킬 실행 시점
- slug: 핵심 키워드 2-4개, kebab-case
  - 한글 허용 (예: `plan-mode-사용법`)
  - 특수문자 제거, 공백→하이픈
  - 최대 30자
  - 동일 파일 존재 시: `-v2`, `-v3` 접미사

저장 형식:

```markdown
# {선택된 제목}

## 배경
{어떤 상황이었는지}

## 과정
{Claude Code를 어떻게 활용했는지}
- 사용한 프롬프트 예시
- 활용한 기능
- 시행착오

## 결과
{before/after 비교, 구체적 숫자}

## 배운 점
{인사이트}
```

```
사례글이 완성됐어요!

📄 저장: my/cases/YYMMDD-{slug}.md

GPTers에 게시할 준비가 됐어요.
복사해서 게시글로 올려주세요!
다음 주에도 /gpters-writer 로 사례글 써주세요! 📝
```

## Rules

- 질문이 길어지면 이탈하므로 3턴 x 4선택지로 빠르게 소재를 수집합니다
- 스터디원 대부분이 비개발자이므로 기술 용어 대신 일상어를 씁니다
- "좋아졌다"보다 "30분→5분"이 설득력 있으므로 구체적 숫자와 변화를 포함합니다
- 실패 경험도 다른 스터디원에게 유용한 학습 자료이므로 가치 있음을 강조합니다
- 세션 스캔은 힌트 제공용이므로 실패해도 사례글 작성 자체에는 영향이 없습니다
- 에러 메시지는 사용자를 불안하게 만들므로 스캔 실패 시 힌트 없이 Turn 1을 바로 시작합니다
