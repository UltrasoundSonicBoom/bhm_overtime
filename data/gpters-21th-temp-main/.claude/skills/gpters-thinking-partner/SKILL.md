# GPTers Thinking Partner

"뭘 만들고 싶은데 정리가 안 돼" → 3턴 대화로 프로젝트 정의서를 만들어줍니다.

## Core Principles

- 스터디원은 뭘 자동화해야 하는지, 뭐가 가능한지 모름
- 한 번에 한 질문, 4개 선택지, 총 3턴으로 끝남
- 질문마다 GPTers 사례로 힌트 제공

## Quick Start

```text
/gpters-thinking-partner             # 처음부터 시작
/gpters-thinking-partner --browse    # 사례 먼저 구경
/gpters-thinking-partner $ARGUMENTS  # 아이디어 직접 입력
```

## Workflow

### Turn 1: 뭘 하고 싶어요?

AskUserQuestion:
- "Claude Code로 어떤 걸 해보고 싶으세요?"
  - 반복 업무 자동화 (매번 같은 일 하는 거 줄이기)
  - 코딩/개발 도우미 (코드 짜거나 고치기)
  - 문서/콘텐츠 만들기 (글 쓰거나 정리하기)
  - 잘 모르겠어요 (사례 보여주세요)

**"잘 모르겠어요" 선택 시:**
`gpters-docs/cases/` 에서 사례 5개를 채팅에 보여준 후 AskUserQuestion:
- "이 중에 해보고 싶은 게 있어요?"
  - {사례 1 제목} → Turn 2로 (해당 카테고리)
  - {사례 2 제목} → Turn 2로
  - {사례 3 제목} → Turn 2로
  - 다른 거 해보고 싶어요 (직접 설명)

### Turn 2: 구체적으로 뭔가요?

Turn 1 답변에 따라 맞춤 4선택지 제시.

**반복 업무 선택 시:**
- "어떤 종류의 반복 업무인가요?"
  - 데이터 정리/변환 (엑셀, CSV 등)
  - 보고서/요약 만들기
  - 알림/공유 자동화 (슬랙, 이메일 등)
  - 기타 (직접 설명)

**코딩/개발 선택 시:**
- "어떤 도움이 필요하세요?"
  - 새 기능 만들기
  - 버그 수정/디버깅
  - 코드 리뷰/리팩토링
  - 기타 (직접 설명)

**문서/콘텐츠 선택 시:**
- "어떤 종류의 글인가요?"
  - 블로그/사례글
  - 보고서/제안서
  - 번역/요약
  - 기타 (직접 설명)

### Turn 3: 지금은 어떻게 하고 있어요?

- "이 작업을 지금은 어떻게 하고 계세요? 시간이 얼마나 걸리나요?"
  - 수동으로, 30분 이상 걸림
  - 수동으로, 금방 끝나지만 매번 귀찮음
  - 반자동 (일부 도구 사용 중)
  - 아직 안 하고 있음 (새로 시작)

## Result: 정의서 미리보기 (채팅 출력)

3턴 답변을 종합하여 채팅에 미리보기:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 프로젝트 정의서 미리보기
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

프로젝트: {자동 생성된 프로젝트 이름}

▸ 한 줄 요약
  {Turn 1+2 기반 요약}

▸ 현재 상태 (AS-IS)
  {Turn 3 기반}

▸ 목표 상태 (TO-BE)
  {Claude Code로 바꿀 모습}

▸ 첫 번째 단계
  {plan mode에서 시작할 작업}

▸ 참고 사례
  {관련 GPTers 사례}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## HITL: 문서화 + 구현 분기

AskUserQuestion:
- "이 정의서로 어떻게 할까요?"
  - 문서로 저장하고 직접 구현할게요 → 저장 후 종료
  - 문서로 저장하고 바로 구현 시작 (plan mode) → 저장 + EnterPlanMode
  - 수정할 부분이 있어요 → 피드백 반영 후 다시 미리보기
  - 처음부터 다시 할래요 → Turn 1로

### 저장 시

`my/projects/YYMMDD-{slug}.md` 에 정의서를 마크다운 헤더(`##`) 형식으로 저장.

### Slug 생성 규칙

파일명: `YYMMDD-{slug}.md`
- YYMMDD: 스킬 실행 시점
- slug: 핵심 키워드 2-4개, kebab-case
  - 한글 허용 (예: `plan-mode-사용법`)
  - 특수문자 제거, 공백→하이픈
  - 최대 30자
  - 동일 파일 존재 시: `-v2`, `-v3` 접미사

### 바로 구현 시

정의서 저장 후 `EnterPlanMode` tool 호출.
정의서 내용이 plan의 초기 컨텍스트로 자동 전달됨:

```
정의서를 저장했어요!
📄 my/projects/YYMMDD-{slug}.md

plan mode로 전환합니다. 정의서 기반으로 구현 계획을 세울게요.
```

## Conversation Style

- 한 번에 한 질문, 4개 선택지
- 답변에 공감 ("아 그거 진짜 귀찮죠!")
- 막막할 때 사례가 구체적 영감을 주므로 "모르겠다"고 하면 사례로 힌트를 제공합니다
- 스터디원 대부분이 비개발자이므로 기술 용어 대신 일상어를 씁니다
- 3턴을 넘기면 피로도가 올라가므로 3턴 + 1 HITL로 완료합니다

## Rules

- 여러 질문을 한꺼번에 던지면 인지 부하가 커지므로 한 번에 한 질문 + 4개 선택지로 진행합니다
- 스터디원 대부분이 비개발자이므로 기술 용어 대신 일상어를 쓰면 이탈률이 낮아집니다
- 실제 사례가 가장 강력한 영감 소스이므로 GPTers 사례를 적극 활용합니다
- 3턴을 넘기면 피로도가 올라가므로 3턴 + 1 HITL로 완료합니다
- 없는 정보를 꾸며내면 신뢰를 잃으므로 사례가 0개일 때 솔직히 안내하고 자유 입력으로 전환합니다
- "기타" 선택은 사용자의 고유한 맥락이므로 자유 입력값을 Turn 2의 맥락으로 활용합니다
