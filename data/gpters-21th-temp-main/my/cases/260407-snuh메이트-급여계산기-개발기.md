# 병원 간호사 급여 계산기 앱을 직접 만들었습니다 — SNUH 메이트 개발기

## 배경

서울대학교병원(SNUH), 보라매병원 직원들이 쓸 수 있는 급여·스케줄 관리 앱이 필요했다.
시간외 근무수당, 야간수당, 가족수당, 명절지원비… 병원 급여 구조는 항목이 복잡해서
엑셀로도 계산하기 벅찬 수준이다. 기성 앱은 이 구조를 전혀 반영하지 못했고,
직접 만드는 건 코딩을 모르면 불가능한 일이었다.

## 과정

Claude Code와 함께 **SNUH 메이트**를 처음부터 만들었다. 커밋 80개, 약 한 달간의 작업이었다.

**주요 작업 흐름:**
- 실제 급여명세서 PDF를 Claude Code에 넘겨서 수식을 역산하고 코드와 비교 검증
- 수당 항목(가족수당·명절지원비·야간수당)을 병원 보수규정에 맞게 하나씩 구현
- 공휴일 외부 API 연동 시 화면이 백지가 되는 버그 → 이중 렌더링 구조로 해결
- Supabase + Google OAuth로 "가족 모드(family mode)" 클라우드 동기화 구현
- Vercel로 실제 배포 (snuhmoney.vercel.app)
- PDF/Excel/CSV 급여명세서 파서로 앱 vs 실제 명세서 비교 기능까지 추가

**특이했던 점:** 코딩보다 "이 계산이 실제 병원 규정과 맞는가"를 검증하는 게 더 많은 시간을
차지했다. Claude Code가 PDF를 직접 분석해서 수식을 검증해줬기 때문에 가능했다.

## 결과

- 코딩 지식 없이 → 실제 배포된 PWA 완성 (모바일 홈 화면 설치 가능)
- 시간외·온콜·휴가 캘린더 + 급여 자동 계산 + 명세서 파서 전부 포함
- 로컬 모드(개인용) / 가족 모드(클라우드 동기화) 투트랙 구조
- 보라매병원·국립소방병원까지 커버하도록 확장

## 배운 점

"와!" 하는 순간은 없었지만, 작업 내내 AI가 옆에 앉아있는 느낌이었다.
특히 실제 급여명세서를 던져주고 "내 코드랑 맞아?"라고 물어볼 수 있는 것 —
이게 가장 큰 차이였다. 혼자였다면 수식 하나 검증하는 데 며칠이 걸렸을 일이
한 세션에 끝났다. Claude Code는 "코딩 도구"가 아니라 "같이 만드는 동료"였다.
