# AI 팀 13명 고용했더니 벌어진 일 — 병원 직원용 앱 개발기

> GPTers 21기 주간 사례글 | 2026-04-12 | kgh1379

## 배경

저는 병원에서 일하는 직원입니다.

매달 "내 초과근무 수당이 맞게 나온 건가?", "연차가 몇 개 남았지?", "퇴직금이 얼마쯤 될까?" 를 계산하려면 엑셀 파일 열고, 규정 PDF 뒤지고, 인사팀에 문의하는 과정을 반복해야 했습니다.

그래서 직접 만들기로 했습니다. **SNUH Mate** — 서울대학교병원 직원을 위한 급여·휴가·시간외 계산 앱.

문제는 제가 원하는 기능들이 "혼자서는 못 만드는" 수준이었다는 겁니다.

- **RAG 기반 규정 챗봇** (조항 번호까지 인용하는 AI 답변)
- **Google Drive 자동 백업 + Calendar 연동**
- **비개발자 운영자가 FAQ를 편집할 수 있는 Admin 시스템**
- **PDF/Excel 명세서를 읽어서 실제 급여를 분석하는 파서**

이것들을 혼자 기획부터 구현까지 하려면... 솔직히 몇 달이 걸리든 끝을 장담할 수 없었습니다.

---

## 과정

Claude Code와 함께 이번 주(4월 5일~12일)에 한 것들:

### 첫째, 앱 기반을 완전히 바꿨습니다

온보딩 페이지를 전면 리디자인하고, 급여 탭을 3탭 구조로 개편하고, 이미지 21MB를 836KB로 줄이고, iOS Safari 버그를 잡고, PWA 설치 아이콘을 달았습니다. 총 133개의 커밋. 혼자 하루에 7시간씩 8일.

"Claude Code 없이는 이걸 언제 했겠냐"는 말이 아닙니다. **이 중에서 상당 부분은 아예 시도조차 못 했을 작업**입니다.

### 둘째, RAG 파이프라인을 실제로 동작시켰습니다

병원 규정 PDF를 청크로 잘라서 → pgvector에 임베딩 저장 → 직원이 "야간수당 어떻게 계산해요?" 물어보면 → 조항 번호와 함께 답변하는 챗봇.

이 구조를 혼자 설계하고 구현한다? Phase 0~4 백엔드를 **33개 파일, 8,483줄로 하루 안에 완성**한 건 Claude Code가 있었기 때문에 가능했습니다.

### 셋째, 그리고 가장 인상적인 순간

3레이어 에이전트 팀을 처음 가동했을 때입니다.

```
빌드팀 (개발)     → 백엔드 API, RAG 파이프라인 구현
운영팀 (비개발자) → FAQ 업데이트, 규정 문서 적재, 검토 큐
사용자팀 (직원)   → 규정 질의, 급여 시뮬레이션, 챗봇 응답
```

총 13개 에이전트가 각자 역할을 갖고 협업하는 구조입니다. 제가 직접 설계한 것도 있지만, "이런 팀이 필요하다"는 기획 의도를 Claude Code와 함께 다듬었더니 에이전트 역할 정의부터 라우팅 로직까지 실제로 돌아가는 시스템이 만들어졌습니다.

혼자 개발자이고 혼자 기획자이고 혼자 운영자인 상황에서, 처음으로 **"팀이 있다"는 느낌**을 받은 순간이었습니다.

## 배운 점

**"못 만든다"와 "혼자 만든다"는 다른 말입니다.**

이번 주 전까지 RAG 파이프라인은 "나중에 팀 생기면 해볼까" 영역이었습니다. 에이전트 팀 구성은 "언젠가 공부해서"였고, Google OAuth는 "문서 읽을 시간 나면"이었습니다.

Claude Code와 함께하니 이것들이 **"이번 주"** 가 됐습니다.

다만 솔직히 남은 약점도 있습니다. 133개 커밋 동안 테스트 코드는 **0줄**이었습니다. 급여 계산이 10번 수정되는 동안 자동 검증이 없었다는 건, 지금 당장은 빠르지만 나중에 사용자에게 잘못된 정보를 줄 수 있는 위험입니다. 다음 주 최우선 과제로 잡아놨습니다.

AI와 함께 빠르게 만들 수 있다는 것, 그리고 그 속도가 안전망 없이는 위험해질 수 있다는 것 — 둘 다 이번 주에 배웠습니다.
