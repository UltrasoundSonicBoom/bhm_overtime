# SNUHmate PII Patterns — 한국 개인정보 마스킹 도메인 지식

이 스킬은 _워크플로우가 아닙니다_. 한국어 PII 식별·마스킹에 필요한 정규식·NER 룰·검증 전략을 모은 **참조 문서**입니다. B6 마스킹 서비스를 만들거나 검토할 때 import 해서 사용.

## 마스킹 토큰 명세

표준화된 치환 토큰 — 모든 모듈에서 동일하게 사용:

| kind    | 한국 명칭      | 토큰           |
| ------- | -------------- | -------------- |
| rrn     | 주민등록번호   | `[RRN]`        |
| frn     | 외국인등록번호 | `[FRN]`        |
| phone   | 전화번호       | `[PHONE]`      |
| email   | 이메일         | `[EMAIL]`      |
| biz_no  | 사업자등록번호 | `[BIZ_NO]`     |
| person  | 인명           | `[PERSON]`     |
| address | 주소           | `[ADDRESS]`    |
| dob     | 생년월일       | `[DOB]`        |
| account | 계좌번호       | `[ACCOUNT]`    |
| card    | 카드번호       | `[CARD]`       |
| patient | 환자등록번호   | `[PATIENT_ID]` |
| medrec  | 의무기록번호   | `[MEDREC]`     |

## 정규식 (한국 PII)

| 항목                 | 정규식                                                                     | 비고                                       |
| -------------------- | -------------------------------------------------------------------------- | ------------------------------------------ | ------ | ------ | ---------------------------------- | --- |
| 주민등록번호         | `\b\d{6}[-\s]?[1-4]\d{6}\b`                                                | 7번째 자리 1~4 (성별·세기)                 |
| 외국인등록번호       | `\b\d{6}[-\s]?[5-8]\d{6}\b`                                                | 7번째 자리 5~8                             |
| 휴대전화             | `\b01[016789][-\s]?\d{3,4}[-\s]?\d{4}\b`                                   |                                            |
| 지역전화             | `\b0(2                                                                     | 3[1-3]                                     | 4[1-4] | 5[1-5] | 6[1-4])[-\s]?\d{3,4}[-\s]?\d{4}\b` |     |
| 사업자번호           | `\b\d{3}[-\s]?\d{2}[-\s]?\d{5}\b`                                          |                                            |
| 이메일               | `[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}`                           | RFC 5322 단순형                            |
| 우편번호 (5자리)     | `\b\d{5}\b` (시·도 인접 시 가중치)                                         | 일반 5자리 숫자와 충돌 — NER 컨텍스트 필수 |
| 환자등록번호         | `\b\d{8}\b` 또는 병원별 포맷                                               | 병원별 다름 — 호스팅 시 재정의             |
| 의무기록번호         | `\bMR[-_]?\d{6,10}\b`                                                      | 가정, 실제 형식 확인 필요                  |
| 신용카드 (Luhn 제외) | `\b(?:\d{4}[-\s]?){3}\d{4}\b`                                              | 4-4-4-4 패턴, 잘못된 매칭 다수 — 검증 필요 |
| 계좌번호             | 은행별 가변 (10~14자리, 하이픈 포함)                                       | 일관된 정규식 어려움 — context-aware 필요  |
| 생년월일 (한글 포함) | `\b\d{4}년\s?\d{1,2}월\s?\d{1,2}일\b` 또는 `\b\d{4}[-./]\d{2}[-./]\d{2}\b` | 다른 날짜와 구분 어려움                    |

## NER 룰 — 인명·주소 (spaCy 미설치 폴백)

### 인명 후보

- 한글 2~4음절, 첫 글자가 흔한 성: `김 이 박 최 정 강 조 윤 장 임 한 오 서 신 권 황 안 송 류 전 홍 고 문 양 손 배 백 허 유 남 심`
- 단, 일반 명사(예: "김치")와 충돌 — 다음 추가 룰:
  - 직책·호칭 인접: `<후보> 환자|선생|간호사|의사|선배|님` → 강한 인명 신호
  - 조사 인접: `<후보>이/가/은/는/을/를/에게/께` → 인명 후보
- 너무 많은 false positive 의심되면 → 폴백을 OFF 하고 LLM 검증 의존

### 주소 후보

- 한글 + 행정 키워드: `<한글>+(특별시|광역시|도|시|군|구|동|읍|면|리|로|길|번지)`
- 우편번호 인접 시 강한 신호
- 건물명·아파트 이름 (예: "삼성래미안") → 일반 폴백으로 잡기 어려움 — LLM 검증

## LLM 검증 전략

세 옵션. 외부 송신 최소화가 우선:

| 옵션                      | 외부 송신 | 정확도 | 복잡도      |
| ------------------------- | --------- | ------ | ----------- |
| 1. Microsoft Presidio     | 0         | 중     | 자가호스팅  |
| 2. OpenAI GPT-4o-mini     | 있음      | 상     | API 키 필요 |
| 3. Anthropic Claude Haiku | 있음      | 상     | API 키 필요 |

**권장:** 옵션 1 (Presidio) 기본, 옵션 2/3 은 환경변수 `SMATE_B6_EXTERNAL_LLM=anthropic|openai` 로 옵트인. 외부에 보낼 땐 _마스킹된 텍스트만_ 보낸다 (원본 PII 외부 송신 0).

### LLM 검증 프롬프트 템플릿

```
다음 텍스트는 1차·2차 마스킹을 거친 결과입니다. 한국 개인정보(PII)가 여전히 남아있는지 검증하세요.

마스킹된 텍스트:
<masked_text>

응답 형식 (JSON):
{
  "passed": true|false,
  "residue": [
    {"kind": "person|address|phone|...", "snippet": "<일부 인용 5자 이내>", "reason": "..."}
  ]
}

- 이미 [PERSON], [RRN] 등 토큰으로 치환된 부분은 검출 대상이 아닙니다.
- 잔존 의심만 보고하세요. 일반 단어를 PII로 추측하지 마세요.
```

## 한계 (v1)

- Excel/PDF/이미지 출력은 v1에서 `.txt` 로 변환됨. 원본 포맷 보존은 v2.
- Presidio analyzer 는 영어 기본 — 한국어 recognizer 는 별도 정의 필요. v1에서는 한국 PII는 정규식+NER이 1차 책임.
- 환자번호·의무기록번호는 병원마다 형식이 다름 — 배포 시 환경변수 `SMATE_B6_PATIENT_ID_REGEX` 로 오버라이드.
