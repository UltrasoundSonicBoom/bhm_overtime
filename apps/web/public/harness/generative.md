# 생성 아웃풋 (Generative Outputs) — 병원 AI 패턴

EMR 데이터와 임상 노트를 기반으로 다양한 의료 문서를 자동 생성하는 패턴.

## 핵심 시나리오

| 문서 유형 | 입력 | 기대 효과 |
|---------|------|---------|
| SOAP 간호기록 | 간단한 메모 + 바이탈 | 기록 작성 50% 시간 단축 |
| 퇴원 요약서 | 입원기간 EMR 데이터 | 전공의 작성 30분 → 5분 |
| 행정 보고서 | 월별 지표 데이터 | 보고서 완성 자동화 |
| 환자 안내문 | 처치·검사 정보 | 인쇄/문자/QR 형식 즉시 |
| 문헌 요약 | PubMed 검색 쿼리 | EBP 근거 10분 내 확보 |

## 적용 직종

- **간호사** (SOAP 기록, 환자 안내문)
- **의사·전공의** (퇴원요약, 의뢰서, 문헌조회)
- **관리자·파트장** (행정보고서)
- **행정사무직** (안내문 발송)

## Claude Code 구현 가이드

```
SOAP 간호기록 자동 생성 패턴:
1. 간호사 짧은 메모 수신 ("37.8도, 호소 두통, 진통제 투여")
2. 바이탈 사인 데이터 EMR에서 자동 수집
3. SOAP 구조로 분류 (S: 주관적, O: 객관적, A: 평가, P: 계획)
4. 표준 간호 용어(NANDA) 적용
5. 간호사 최종 검토 후 EMR 등록

퇴원 요약서 생성 패턴:
1. 입원기간 주요 이벤트 시간순 추출
2. 주진단·부진단·처치·결과 요약
3. 퇴원 후 관리 지침 생성
4. 의사 검토 게이트 후 확정
```

## 주요 기술 스택

- **문서 생성**: Claude 3.5 Sonnet (한국어 의료 문서 최적화)
- **EMR 연동**: HL7 FHIR API (표준 인터페이스)
- **PubMed 검색**: NCBI E-utilities API
- **다형식 출력**: PDF/HTML/SMS/QR 코드

## 품질 관리

- 생성된 문서는 반드시 담당자 검토 후 확정
- 생성 문서에 "AI 초안" 워터마크 표시
- 오류 발생 시 원본 데이터 보존
