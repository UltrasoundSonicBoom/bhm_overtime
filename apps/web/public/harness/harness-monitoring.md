# 병동 지표 모니터링 팀 하네스

병동 핵심 안전 지표를 실시간으로 수집·분석하고 임계값 초과 시 즉각 에스컬레이션한다.

## 에이전트 구성

```
🧠 오케스트레이터 (상시 대기)
    │
    ├── 📥 데이터수집 에이전트
    │     EMR·간호 기록·사고 보고 시스템 연결
    │     지표 데이터 15분 주기 수집
    │
    ├── 📊 패턴분석 에이전트
    │     추세 분석 (이동평균·이상치 감지)
    │     병원 평균·벤치마크 대비 비교
    │     원인 요인 탐색
    │
    └── 🚨 알림발송 에이전트
          임계값 초과 → 담당자 즉시 알림
          긴급도 분류 (즉각 대응 / 모니터링 강화)
```

## 모니터링 지표

| 지표 | 임계값 | 알림 대상 |
|-----|-------|---------|
| 낙상 발생률 | > 병원 평균 1.5배 | 병동 관리자 + 안전팀 |
| 병원 감염률 | 주간 증가 추세 | 감염관리팀 |
| 욕창 발생률 | > 기준치 | 상처간호팀 |
| 30일 재입원 | > 15% | 케어매니지먼트 |
| 초과근무 시간 | 월 30h↑ 직원 | 인사팀 |

## 실행 흐름

```
상시 모니터링 → 이상 감지 시:
1. 패턴분석 에이전트: 원인 분석 보고서 생성
2. 알림발송 에이전트: 담당자 알림 + 권고 조치
3. 오케스트레이터: 14일 후 개선 여부 자동 추적
```

## 적용 직종

- **관리자·파트장** (병동 안전 관리 책임)
- **감염관리팀** (감염 지표)
- **인사팀** (직원 소진 지표)

## SNUHmate 연동

- 초과근무 계산 탭 데이터 → 직원 소진 지표 소스
- 향후 관리자용 대시보드 탭 연동 예정
