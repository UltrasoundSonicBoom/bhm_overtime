BHM Overtime 백엔드 아키텍처 업그레이드 계획
Context
현재 BHM Overtime(SNUH Mate)은 Vanilla JS SPA + localStorage + 선택적 Supabase 동기화로 운영 중이다. 병원 노동조합 규정(3개 PDF, ~30페이지)이 data.js에 하드코딩되어 있어, 매년 규정 변경 시 코드를 직접 수정·배포해야 한다.

목표: RAG 챗봇, DB 기반 규정 관리, FAQ/QNA, Admin 페이지를 추가하여 규정을 코드 배포 없이 관리하고, 직원들이 자연어로 규정을 검색할 수 있게 한다.

기술 스택: Drizzle ORM + Supabase (PostgreSQL + pgvector) + Hono (API 서버) + OpenAI Embeddings

3. pgvector 설정 (Supabase SQL Editor)
CREATE EXTENSION IF NOT EXISTS vector;

-- HNSW 인덱스 (코사인 유사도)
CREATE INDEX idx_reg_docs_embedding ON regulation_documents
  USING hnsw (embedding vector_cosine_ops) WITH (m = 16, ef_construction = 64);

CREATE INDEX idx_faq_embedding ON faq_entries
  USING hnsw (embedding vector_cosine_ops) WITH (m = 16, ef_construction = 64);

-- 유사도 검색 함수
CREATE OR REPLACE FUNCTION match_regulation_chunks(
  query_embedding vector(1536),
  match_threshold float DEFAULT 0.7,
  match_count int DEFAULT 5
)
RETURNS TABLE (id int, section_title text, content text, similarity float)
LANGUAGE plpgsql AS $$
BEGIN
  RETURN QUERY
  SELECT rd.id, rd.section_title, rd.content,
         1 - (rd.embedding <=> query_embedding) AS similarity
  FROM regulation_documents rd
  WHERE 1 - (rd.embedding <=> query_embedding) > match_threshold
  ORDER BY rd.embedding <=> query_embedding
  LIMIT match_count;
END;
$$;
4. RAG 챗봇 파이프라인
흐름
사용자: "온콜 출근하면 수당 얼마?"
  ↓
1. OpenAI text-embedding-3-small로 쿼리 임베딩
  ↓
2. 병렬 검색:
   a) regulation_documents 벡터 검색 (top 5, threshold 0.7)
   b) faq_entries 벡터 검색 (top 3, threshold 0.75)
  ↓
3. FAQ 최상위 매치 > 0.92이면 → FAQ 답변 직접 반환 (LLM 호출 안 함)
  ↓
4. 아니면 → 규정 청크 + FAQ를 컨텍스트로 gpt-4o-mini 호출
  ↓
5. 응답 + 출처(조항 번호) 반환
PDF 청킹 전략
조항 경계(제32조, 제34조 등)로 1차 분할
500 토큰 초과 시 문단 단위로 2차 분할, 50 토큰 오버랩
메타데이터: { sectionTitle, articleRef, category, page }
총 예상 청크: 100~150개 (비용 ~$0.01)
핵심 코드: server/src/services/rag.ts
export async function ragSearch(query: string) {
  // 1. 쿼리 임베딩
  const embedding = await openai.embeddings.create({
    model: 'text-embedding-3-small', input: query
  });
  const vec = `[${embedding.data[0].embedding.join(',')}]`;

  // 2. 병렬 벡터 검색
  const [regChunks, faqMatches] = await Promise.all([
    db.execute(sql`
      SELECT id, section_title, content,
             1 - (embedding <=> ${vec}::vector) as similarity
      FROM regulation_documents
      WHERE 1 - (embedding <=> ${vec}::vector) > 0.7
      ORDER BY embedding <=> ${vec}::vector LIMIT 5
    `),
    db.execute(sql`
      SELECT id, question, answer, article_ref,
             1 - (embedding <=> ${vec}::vector) as similarity
      FROM faq_entries WHERE is_published = true
        AND 1 - (embedding <=> ${vec}::vector) > 0.75
      ORDER BY embedding <=> ${vec}::vector LIMIT 3
    `),
  ]);

  // 3. 고신뢰 FAQ 매치 → 직접 반환
  if (faqMatches.rows[0]?.similarity > 0.92) {
    return { answer: faqMatches.rows[0].answer, isFaqMatch: true };
  }

  // 4. LLM 호출
  const context = regChunks.rows.map(r => `[${r.section_title}]\n${r.content}`).join('\n---\n');
  const completion = await openai.chat.completions.create({
    model: 'gpt-4o-mini',
    messages: [
      { role: 'system', content: 'SNUH 병원 HR 어시스턴트. 규정 근거만으로 답변. 한국어.' },
      { role: 'user', content: `규정:\n${context}\n\n질문: ${query}` }
    ],
    temperature: 0.3, max_tokens: 800,
  });

  return { answer: completion.choices[0].message.content, sources: regChunks.rows };
}

// 기존 Supabase JWT 토큰을 검증 (supabaseClient.js와 동일한 인증 체계)
export const authMiddleware = createMiddleware(async (c, next) => {
  const token = c.req.header('Authorization')?.substring(7);
  const { data: { user } } = await supabase.auth.getUser(token);
  if (!user) return c.json({ error: 'Unauthorized' }, 401);
  c.set('user', user);
  await next();
});

// admin_users 테이블에서 역할 확인
export const adminMiddleware = createMiddleware(async (c, next) => {
  const user = c.get('user');
  const [admin] = await db.select().from(adminUsers)
    .where(eq(adminUsers.userId, user.id)).limit(1);
  if (!admin?.isActive) return c.json({ error: 'Forbidden' }, 403);
  await next();
});
