# SNUHmate 페르소나 매트릭스

`smate-feature-ship` Step 1 페르소나 게이트의 ground truth. 각 B 슬롯의 실제 사용자 인터뷰 결과를 여기에 기록한다.

> **규칙:** 이 파일의 행이 비어 있으면 해당 슬롯의 기능 구현 시작 불가. 추측으로 채우지 말 것. B1 폐기 교훈: "야간 1번 더 서면 명절수당 기준 충족" 같은 병원 현실과 맞지 않는 기능은 처음부터 잘못된 것.

## 행 형식

```
| B# | 기능명 | 대상 직종 | 핵심 페인포인트 | 현재 해결 방법 | 기대 효과 | 인터뷰 날짜 |
```

## 인터뷰 가이드

페르소나 행을 채우기 위한 최소 질문:

1. **직종·부서**: 어떤 직종인가? (간호직 / 운영기능직 / 일반직 / 시설직 등)
2. **페인포인트**: 이 기능이 없어서 지금 어떤 불편이 있나?
3. **현재 해결 방법**: 지금은 어떻게 해결하고 있나? (엑셀, 카톡 공지, 수기 등)
4. **기대 효과**: 이 기능이 생기면 어떻게 달라지나?
5. **사용 빈도**: 얼마나 자주 쓸 것 같나?

인터뷰 후 위 표에 기록하고, `docs/harness/b-catalog.md` 상태를 업데이트한다.
