# 카드뉴스·숏폼 파이프라인

## 목표

- 고정 카테고리 5개: `서울대학교병원`, `의료AI`, `AI 활용`, `연말정산`, `재테크`
- 수집 주기: 48시간
- 1회 수집량: 카테고리당 최신 원문 3건, 총 15건
- 결과물:
  - 모바일 카드뉴스 5장
  - 인스타 숏폼용 하이라이트 10개
  - Remotion 일괄 렌더용 JSON manifest

## 파이프라인

1. 원천 데이터 수집
   - Google News RSS
   - OpenAlex 공개 초록
   - 최근 2일 자료를 우선 수집하고, 부족하면 직전 14일 자료로 보강

2. 맥락 추출
   - 제목, 날짜, 발표 주체, 공개 초록에서 지금 달라진 점 추출
   - 기사/논문을 섞어 카테고리별 3건만 선별

3. 추천/생성
   - 카드뉴스는 `표지 -> 헤드라인 흐름 -> 채택 원문 -> 연구/배경 맥락 -> 다음 체크포인트`
   - 숏폼은 `핵심 포인트 추출 -> 후킹 제목 수정 -> 1080x1920 템플릿 적용`

4. 바로 소비 가능한 형태로 배포
   - `cardnews.html`: 모바일 카드뉴스 브라우징
   - `shorts-studio/data/latest-shorts.json`: Remotion 입력 데이터
   - `shorts-studio/out/clip-01.mp4 ~ clip-10.mp4`: 최종 숏폼 렌더 결과

## 카드뉴스 프롬프트

```text
최근 48시간 안에 공개된 자료를 우선 수집하고, 부족하면 직전 14일 안의 자료로 3건까지 보강한다.
기사와 논문은 중복 주장을 피하고 서로 보완되는 맥락으로 묶는다.
모바일 카드뉴스는 표지, 헤드라인 흐름, 채택 원문, 연구 또는 배경 맥락, 다음 체크포인트 5장으로 구성한다.
문장은 짧고 선명하게 유지하고, 카드마다 원문 연결은 최대 3개만 붙인다.
```

## 숏폼 프롬프트

```text
긴 원문에서 먼저 핵심 포인트를 추출한다.
그다음 숏폼 단독 시청자가 멈춰볼 후킹 제목으로 다시 쓴다.
마지막으로 1080x1920 세로 템플릿에 맞춰 상단 제목, 중앙 핵심, 하단 출처를 배치한다.
10개 결과물을 동일한 템플릿으로 일괄 렌더한다.
```

## 레이아웃 기준

- 카드뉴스
  - 모바일 스와이프용 4:5 카드
  - 첫 장에서 바로 요지를 읽을 수 있는 큰 제목
  - 카드별 색 대비를 분명히 나눠 스캔 속도 확보

- 숏폼
  - 1080x1920
  - 상단 브랜드/카테고리
  - 중앙 후킹 제목 + 핵심 2줄
  - 하단 출처 밴드 + 진행 바

## GPters 패턴 반영

- 업무자료 선별 패턴: `수집 -> 맥락 추출 -> 추천/생성 -> 바로 소비`
- Remotion 숏폼 패턴:
  - 하이라이트 추출
  - 제목 수정
  - 템플릿 시각화
  - 여러 결과물 일괄 출력
