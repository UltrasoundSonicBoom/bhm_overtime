## 발견 버그 및 처리

### BUG-001 — 수정 완료 ✅
**날짜 미선택 시 NaN-NaN 키로 데이터 저장**

- **파일**: [app.js](../app.js) `saveOtRecord()` (line 2119)
- **현상**: 날짜 패널을 닫은 상태에서 저장 버튼 접근 가능 → `dateStr = ""` → localStorage에 `NaN-NaN` 키로 저장됨
- **영향**: 잘못된 데이터 누적, 기록 조회 불가, 집계 오류
- **수정**: `dateStr` 빈값 검사 후 `alert('날짜를 먼저 선택해주세요.')` 반환 처리
- **커밋**: `fix(qa): 날짜 미선택 저장 방지 + 프로필 저장 시 연차 즉시 반영`

### BUG-002 — 수정 완료 ✅
**프로필 저장 후 잔여 연차 미즉시 갱신**

- **파일**: [app.js](../app.js) `saveProfile()` (line 299)
- **현상**: 입사일을 포함한 프로필 저장 시 `applyProfileToLeave()` 호출 누락 → 휴가 탭에서 "프로필 입사일 설정 필요" 상태 유지
- **영향**: 연차 잔여일 추적 불가, 사용자가 탭 이동/새로고침 해야 반영됨
- **수정**: `saveProfile()` 종료 시 `applyProfileToLeave()` 자동 호출 추가
- **커밋**: 위 동일

### BUG-003 — 설계 이슈 (의도적 비활성)

### 현재 구조

```
사용자가 캘린더 날짜 클릭
  → onOtDateClick(year, month, day) 호출
  → otInputPanel.dataset.date = 'YYYY-MM-DD' 설정
  → 저장하기 버튼 노출

패널이 닫힌 상태 (closeOtBottomSheet)
  → otInputPanel.dataset.date = '' 로 초기화
  → 저장하기 버튼은 여전히 DOM에 존재 (display:none 아님)
```

### 제안 1: 저장 버튼 disabled 처리 (권장 — 가장 간단)

날짜가 없으면 버튼 자체를 비활성화. 사용자가 왜 안 되는지 직관적으로 알 수 있음.

```javascript
// onOtDateClick() 내부 — 날짜 선택 시 활성화
document.getElementById('otSaveBtn').disabled = false;

// closeOtBottomSheet() 내부 — 패널 닫힐 때 비활성화
document.getElementById('otSaveBtn').disabled = true;
```

```css
/* style.css */
#otSaveBtn:disabled {
  opacity: 0.4;
  cursor: not-allowed;
}
```

- 장점: 진입 차단이라 alert 없이도 의도 명확
- 단점: 없음

### 제안 4: 저장 시도 시 캘린더로 포커스 이동 (현재 수정 방식 개선)

현재는 `alert()`으로 끝남. alert 대신 캘린더를 하이라이트하고 스크롤하면 UX가 자연스러움.

```javascript
// saveOtRecord() 내 dateStr 검증 부분
if (!dateStr) {
  // 캘린더로 스크롤 후 시각적 힌트
  const cal = document.querySelector('.mini-cal-grid');
  if (cal) {
    cal.scrollIntoView({ behavior: 'smooth', block: 'center' });
    cal.classList.add('shake-hint'); // CSS 애니메이션
    setTimeout(() => cal.classList.remove('shake-hint'), 600);
  }
  return;
}
```

```css
@keyframes shake {
  0%, 100% { transform: translateX(0); }
  25% { transform: translateX(-4px); }
  75% { transform: translateX(4px); }
}
.shake-hint { animation: shake 0.3s ease 2; }
```

- 장점: alert 없이 시각적으로 안내, 모바일에서 자연스러움
- 단점: CSS 추가 필요
