# Phase 3: onclick 위임 마이그레이션 SPEC

> 작성: 2026-04-27
> 동인: Phase 2-regression 의 64개 window 노출 누락 fragility 영구 해결

## 목표

1. 모든 inline `onclick=` (HTML + JS 동적 markup) → `data-action` 속성 + 부모 위임
2. `window.X = X` 호환층 113라인 제거 (KEEP 항목 별도 분류)
3. vercel.json `Content-Security-Policy: script-src 'self'` 적용 → XSS 방어

## 표준 패턴

### Before (inline onclick)
```html
<button onclick="saveProfile()">저장</button>
<button onclick="PAYROLL._otStep('id', 0.25, 'overtimeCalc')">+</button>
```

### After (data-action + delegation)
```html
<button data-action="saveProfile">저장</button>
<button data-action="payrollOtStep"
        data-payroll-id="id"
        data-payroll-delta="0.25"
        data-payroll-section="overtimeCalc">+</button>
```

```js
// shared-utils.js 의 헬퍼
import { registerActions } from './shared-utils.js';

registerActions({
  saveProfile: (el, e) => saveProfile(),
  payrollOtStep: (el, e) => PAYROLL._otStep(
    el.dataset.payrollId,
    parseFloat(el.dataset.payrollDelta),
    el.dataset.payrollSection,
  ),
});
```

## 데이터 속성 컨벤션

- `data-action`: 함수 이름 (camelCase)
- `data-{module}-{param}`: 함수 인자 (예: `data-payroll-id`)
- 인자 없는 함수는 `data-action` 만
- 인자가 숫자면 `parseFloat` / `parseInt` 변환
- 모듈 이름은 짧게 (`payroll`, `lv`, `ot`, `ret` 등)

## 헬퍼 API (shared-utils.js)

| 함수 | 용도 |
|------|------|
| `delegateActions(root, handlers)` | 단일 root 의 click 위임 (정적) |
| `delegateInput(root, handlers)` | input 이벤트 위임 (`data-input-action`) |
| `registerActions(handlers)` | 전역 handlers 동적 추가 (모듈별 등록 표준) |
