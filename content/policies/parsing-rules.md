# 규정 PDF 파싱 규칙 (Korean Collective Agreement PDF Parsing Rules)

> 이 파일은 `2026_조합원_수첩_최종파일.pdf` 및 동일 형식의 단체협약 PDF를 파싱할 때 적용되는 구조 규칙을 정의한다.
> `server/scripts/ingest-regulation-docs.ts`에서 직접 참조한다.

---

## 1. 비내용 페이지 (Skip Pages)

다음 페이지는 실제 규정 내용이 아니므로 파싱에서 제외한다:

| 페이지 | 내용 | 판별 기준 |
|--------|------|-----------|
| 1 | 표지 (Cover) | 제목/조합 이름만 있음 |
| 2 | 발간사 / 전국공공운수노동조합 소개 | 조문 없음 |
| 3 | 주소록 (Address Book) | 전화번호 목록 형태 |
| 4 | 목차 (Table of Contents) | "제N장 …… N" 형태 반복 |
| 5 | "단체협약이란?" 소개 페이지 | 설명 텍스트만 있음 |

**구현:** `SKIP_PAGES = new Set([1, 2, 3, 4, 5])` — pageNumber가 이 집합에 속하면 lines 자체를 건너뜀.

## 2. 문서 계층 구조 (Document Hierarchy)

```
단체협약 (Document)
├── 장 (Chapter)  ← 카테고리 메타데이터; 별도 article로 저장 안 함
│   └── 조 (Article)  ← 청킹 단위 (1개 section = 1개 조)
│       ├── 항 (Paragraph)  ← ①②③ 또는 (1)(2)(3) 마커
│       └── 호 (Sub-item)  ← 가. 나. 다. 또는 1) 2) 3)
└── 부칙 (Supplementary Provisions)  ← 조와 동일하게 청킹
```

**핵심 원칙:**
- **장(Chapter)** 는 섹션 타이틀이 되어서는 안 된다. `chapterTitle` 메타데이터로만 기록한다.
- **조(Article)** 가 청킹의 기본 단위다. 각 조가 하나의 `section`이 된다.
- 조의 내용은 해당 조의 모든 항과 호를 포함한다 (다음 조가 나올 때까지).

## 3. PDF 텍스트 아이템 가르침 (Raw Item Garbling)

pdfjs-dist가 추출하는 텍스트 아이템은 글리프 단위로 분리될 수 있다.
이 PDF의 핵심 특성:

### 3.1 조 제목 분리 패턴

PDF 인코딩에서 조 제목이 다음과 같이 분리된다:

```
# 실제 PDF 아이템 (x좌표 기준 정렬)
x=40  "제 조 목적"      ← 조 번호 자리가 비어 있는 pre-combined 문자열
x=50  "1"               ← 조 번호가 별도 아이템으로 분리됨
x=66  "("
x=90  ")"

# 2px gap threshold 적용 후 row 재조합 결과
"제 조 목적 1 ( )"     ← 잘못된 결과

# 올바른 결과
"제1조 목적( )"
```

다른 패턴 예시:
```
x=40 "제 조 조합"  x=50 "8"  x=60 "(" x=70 "활동의" x=80 "보장" x=90 ")"
→ "제 조 조합 8 ( 활동의 보장 )"  ← 잘못됨
→ "제8조 조합(활동의 보장)"        ← 올바름

x=40 "제"  x=50 "17"  x=62 "조 조합소개"  x=72 "("
→ "제 17 조 조합소개 ("             ← 잘못됨
→ "제17조 조합소개("                ← 올바름
```

### 3.2 장 제목 분리 패턴

```
x=40 "제"  x=50 "1"  x=60 "장"  x=70 "총칙"
→ "제 1 장 총칙"    ← gap이 있어 공백 삽입됨
→ "제1장 총칙"      ← 올바름 (단순 공백 제거)
```

## 4. 정제 함수 규칙 (cleanKoreanLegalHeading)

row 재조합 후 각 line에 대해 다음 순서로 정제를 적용한다:

### Step 1: "제 N 장/조" 형태 → 공백 제거

```regexp
/^제\s+(\d+)\s+(장|조)/
→ "제$1$2"
```

예: `"제 1 장 총칙"` → `"제1장 총칙"`, `"제 17 조 조합소개"` → `"제17조 조합소개"`

### Step 2: "제 조 [title]N" 형태 → 번호를 "조" 앞으로

```regexp
/^제\s+조\s+(.+?)(\d+)(.*)/
→ "제$2조 " + $1.trimEnd() + $3
```

예:
- `"제 조 목적1 ( )"` → `"제1조 목적( )"` (gap ≤ 2px, 공백 없이 붙음)
- `"제 조 목적 1 ( )"` → `"제1조 목적( )"` (gap > 2px, 공백 있음)

단, Step 1 이후 `"제N조"` 형태가 이미 완성된 경우 Step 2 불필요.

### Step 3: 괄호 내부 불필요한 공백 제거

```regexp
/\(\s+/g → "("
/\s+\)/g → ")"
```

예: `"( 활동의 보장 )"` → `"(활동의 보장)"`

### Step 4: 연속 공백 단일 공백으로

```regexp
/\s{2,}/g → " "
```

## 5. 헤딩 판별 규칙 (isPdfHeading)

row 정제 후 다음 정규식으로 헤딩 여부를 판별한다:

```regexp
/^(제\d+장|제\d+조(?:의\d+)?(?:\([^)]*\))?|부칙|별표|\[[^\]]+\])/
```

**주의:** 정제(cleanKoreanLegalHeading) **이후** 에 적용해야 한다.
정제 전 garbled 형태(`"제 조 목적 1"`)는 이 regex에 매치되지 않는다.

## 6. 개정 마커 (Amendment Markers)

```
형태: <YYYY.MM>
예:   <2009.10>  <2015.03>  <2023.09>

위치: 조 제목 뒤 또는 항 텍스트 내 인라인
의미: 해당 조항이 해당 날짜에 개정됨

파싱: versionMarkers 배열에 수집 (기존 extractVersionMarkers 함수 유지)
```

---

## 7. 항 / 호 마커 (Paragraph / Sub-item Markers)

```
항 마커:
  ① ② ③ … (원문자, Unicode circled numbers)
  (1) (2) (3) … (괄호 숫자)

호 마커:
  가. 나. 다. 라. …
  1) 2) 3) …

단서 조항:
  "다만," 으로 시작하는 문장
```

항/호는 청킹 단위를 바꾸지 않는다. 동일 조의 내용으로 유지한다.

---

## 8. 조의절 (Sub-article) 형태

```
형태: 제N조의M
예:   제18조의2 (조합 전임자)

정규식: /제(\d+)조의(\d+)/
article_ref: "제18조의2"
```

---

## 9. 청킹 단위 결정 규칙

| 라인 타입 | 처리 |
|-----------|------|
| 비내용 페이지 | 전체 스킵 |
| `제N장 ...` | `chapterTitle` 메타데이터 업데이트, flush 하지 않음 (조 없이 장만 있으면 빈 섹션 무시) |
| `제N조 ...` | 이전 조 flush → 새 섹션 시작 |
| `부칙` | 이전 조 flush → 새 섹션 시작 |
| `별표[...]` | 이전 조 flush → 새 섹션 시작 |
| 그 외 | 현재 섹션 body에 추가 |

## 10. 섹션 메타데이터 스키마

각 섹션(조) 저장 시:

```typescript
{
  sectionTitle: "제1조 목적( )",      // 정제된 조 제목
  content: "...",                      // 조의 전체 내용 (항/호 포함)
  metadata: {
    source_type: 'pdf',
    chapter_title: "제1장 총칙",       // 속한 장 제목
    article_ref: "제1조",              // 조 번호만
    page_start: 6,
    page_end: 6,
    version_markers: ["<2009.10>"],    // 개정 마커
  }
}
```

## 11. 검증 체크리스트

파싱 완료 후 다음을 확인한다:

- [ ] 파일명(`2026 nojo.pdf` 등)이 article title로 저장되지 않음
- [ ] 제N장 헤딩이 별도 article로 저장되지 않음 (chapter_title 메타데이터로만 존재)
- [ ] `제1조`, `제2조`, ... 가 올바른 형태로 저장됨 (숫자가 제 뒤에 붙음)
- [ ] 괄호 내부 공백 없음: `(활동의 보장)` not `( 활동의 보장 )`
- [ ] 단체협약 전문 내용이 50자 이상인 섹션들만 저장됨
- [ ] 비내용 페이지(1-5) 내용이 저장되지 않음
