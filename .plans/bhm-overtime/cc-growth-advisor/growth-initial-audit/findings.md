## 요약

SNUH 메이트는 온보딩 디자인과 콘텐츠 품질이 이미 높은 수준이다. Neo brutalist 스타일은 병원 게시판에서 눈에 띄는 차별점이 된다. 그러나 D-13 홍보 전에 반드시 해결해야 할 friction이 3가지 있다: (1) 베타 경고문이 잠재 사용자를 쫓아낸다, (2) CTA 문구 "데모"가 진입 장벽, (3) 테스티모니얼이 신뢰를 깎는다.

## 1. 랜딩/온보딩 분석

**강점**
- 헤드라인 "다음 달 월급 얼마 들어올까??" — 직접적이고 강한 타깃 공감형 카피
- 섹션별 Pain tag (사용자 언어 그대로) — "야간 수당, 계산대로 들어온 건지 알 수가 없어요"
- 개발자 인용구 — "평소에 이런게 있으면 좋겠다했던 것들을 만들었습니다. 저만 그랬나요?" — 인간화 탁월

**[CRITICAL] 베타 경고문 2개가 첫인상을 망친다**
- 위치 1: onboarding.html L743-764 — 검은 배경+노란 티커 "테스터 목적외 데이터 저장 금지"
- 위치 2: onboarding.html L812-833 — Hero 내부 "⚠️ 테스터 공지 / 정식 오픈 전입니다"
- shared-layout.js L162-177에서도 동일 티커 렌더
- 홍보일(4/30)은 정식 오픈(5/1) 하루 전 — 베타 딱지를 유지할 이유 없음
- 개선안: D-30 기준으로 경고 박스 제거. 티커는 "🎉 2026.5.1 정식 오픈" 등 긍정 메시지로 교체

**[HIGH] Hero 타이틀 위 불필요한 공백 br 태그**
- onboarding.html L784: `<h1 class="hero-title reveal d1"><br>다음 달 월급<br>` — 첫 br이 모바일 공간 낭비
- 개선안: 첫 `<br>` 제거

## 2. CTA 최적화

**[HIGH] "데모" 표현이 진지한 사용자 진입 장벽**
- 현재: `onboarding.html` L1345-1346 — "🎮 데모 페이지에서 미리 체험하기 →"
- "데모" = "아직 진짜 아님", "🎮" = 게임 이미지 — 병원 직원 맥락과 불일치
- 개선안 A: "내 수당 지금 계산해 보기 →"
- 개선안 B: "로그인 없이 바로 써보기 →"

**[HIGH] 데모 진입 후 핵심 기능까지 4-5단계**
- 현재 CTA 링크: `./index.html?app=1&demo=1` → 홈 탭 랜딩
- 개선안: `./index.html?app=1&demo=1&tab=overtime` → 시간외 탭 직행, 2-3단계로 단축

**[MEDIUM] 튜토리얼 버튼 시각적 계층 너무 낮음**
- onboarding.html L1363-1365: border/shadow 없는 밑줄 텍스트
- 비기술 사용자에게 중요한 진입점인데 거의 안 보임

## 3. 기능 발견성

**[HIGH] "info" 탭 레이블이 한국 사용자에게 의미 없음**
- shared-layout.js L310: `{ icon: '👤', text: 'info', tab: 'profile', ... }`
- 프로필 입력이 수당 자동 계산의 전제조건인데 탭명이 "info"로는 유입 없음
- 개선안: "내정보" 또는 "내설정"으로 변경

**[MEDIUM] 핵심 기능(시간외)이 3번째 탭에 위치**
- 탭 순서: 홈 → 휴가 → 시간외 → 급여 → 규정 → info
- Hero 카피는 월급 계산 강조인데 앱 진입 시 홈 탭이 기본값
- 개선안: 데모 모드 진입 시 시간외 탭 자동 선택

**[LOW] 퇴직 시뮬레이션이 3단계 중첩으로 숨겨짐**
- 급여탭 → 퇴직금 서브탭 → 시뮬레이션 탭 — 고가치 기능인데 발견하기 어려움
